## Project Description

The **Portfolio** project is a personal portfolio website designed to showcase the skills, achievements, and projects of an individual. Built with React and Vite, this application provides an engaging user interface that highlights various aspects of the user's professional journey. The portfolio includes sections for skills, projects, and achievements, making it a comprehensive presentation of the user's capabilities.

### Key Features
- **Responsive Design**: The portfolio is designed to be fully responsive, ensuring a seamless experience across devices.
- **Dynamic Content**: Utilizes JSON files to manage and display project and skill data dynamically.
- **Interactive Components**: Features interactive components such as project cards and skill displays to engage visitors.
- **Smooth Navigation**: Implemented routing for easy navigation between different sections of the portfolio.

## Installation Instructions

### Prerequisites
- Node.js (v14 or later)
- npm (comes with Node.js)

### Step-by-Step Installation Guide
1. **Clone the repository**:
   ```bash
   git clone https://github.com/Scrat1313/Portfolio.git
   cd Portfolio
   ```

2. **Install dependencies**:
   ```bash
   npm install
   ```

3. **Run the development server**:
   ```bash
   npm run dev
   ```

4. **Open your browser** and navigate to `http://localhost:3000` (or the port specified in the terminal) to view the portfolio.

## Usage

To run the project, follow the installation instructions above. Once the development server is running, you can interact with the portfolio in your web browser. 

### Configuration
No additional configuration is required. The project is ready to use out of the box.

## Project Structure

The project structure is organized to facilitate easy navigation and maintenance:

```
Portfolio/
├── public/                  # Static files served by the application
│   ├── files/               # Contains CV and other downloadable files
│   ├── favicon.ico          # Favicon for the website
│   └── vite.svg             # Vite logo
├── src/                     # Main source code for the application
│   ├── assets/              # Static assets like images and files
│   │   ├── files/           # Additional files (e.g., CV)
│   │   ├── images/          # Image assets used in the portfolio
│   ├── components/          # Reusable React components
│   │   ├── commons/         # Commonly used components
│   │   └── layout/          # Layout components (Header, Footer)
│   ├── data/                # JSON files containing data for projects and skills
│   ├── hooks/               # Custom React hooks
│   ├── pages/               # Page components for routing
│   ├── routes/              # Application routing
│   ├── styles/              # CSS styles for the application
│   ├── App.jsx              # Main application component
│   └── main.jsx             # Entry point for the React application
├── .gitignore               # Git ignore file
├── eslint.config.js         # ESLint configuration
├── index.html               # Main HTML file
├── package-lock.json        # Lock file for npm dependencies
├── package.json             # Project metadata and dependencies
├── README.md                # Project documentation
└── vite.config.js           # Vite configuration file
```

### Main Files
- **`App.jsx`**: The root component that renders the application.
- **`main.jsx`**: The entry point for the React application.
- **`routes/routes.js`**: Defines the routes for navigating between different pages.

## Branch Naming Convention

When creating branches, please follow these naming conventions:

- **Feature branches:** `feature/your-feature-name`
- **Bugfix branches:** `fix/your-bugfix-name`
- **Hotfix branches:** `hotfix/your-hotfix-name`

### Examples
- `feature/add-contact-form`
- `fix/hero-design-issue`
- `hotfix/fix-footer-responsive`

## Commit Message Guidelines

We follow the Conventional Commits specification for commit messages. Please use the following format:

```
type(scope): subject
```

### Types
- `fix`: A bug fix
- `feature`: A new feature
- `design`: Changes related to design

### Examples
- `fix(hero): design updated`
- `feature(skills): added new skills section`
- `design(Header): updated layout`

## Development Workflow

1. **Fork and Clone**: Fork the repository and clone it to your local machine.
2. **Create a Feature Branch**: Create a new branch for your feature or bug fix.
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make Commits**: Make your changes and commit them with appropriate messages.
4. **Open a Pull Request**: Push your branch to your forked repository and open a pull request against the `main` branch of this repository.

## Code Style and Standards

- Follow best practices for JavaScript, CSS, and HTML.
- Ensure your code is clean and well-commented.
- Test your changes before submitting a pull request.

## Pull Request Process

1. Ensure your branch is up to date with the `main` branch.
2. Open a pull request and provide a clear description of your changes.
3. The project maintainer will review your pull request. Be open to feedback and make necessary adjustments.

## Issue Reporting

To report bugs or request features, please open an issue in the repository. Be sure to provide as much detail as possible, including steps to reproduce the issue or a description of the feature you’d like to see.
